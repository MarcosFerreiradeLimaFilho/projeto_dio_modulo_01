Comunicação Client x Server

História da Web
(voltar)

A história da web começa com o início da comunicação através do telégrafo e vai evoluindo através dos computadores da época, passando pela guerra fria até chegar em 1989 , com Tim Berners-Lee que dá um passo importante ao unir hipertexto com TCP/IP e cria a World Wide Web, que também é o nome do primeiro navegador.

O que são Clients
(voltar)

Client é quando algum dispositivo consome informação de algum lugar, dentro da internet Cliet seria o dispositivo que consomem informações da internet.

Exemplos: Notebook, Pc 's, Smartphone eles se conectam com outros servidores usando a internet como meio de comunicação.

E como isso funciona? Quando acessamos algum site, o navegador entra na internet , acessa o servidor e retorna com vários arquivos que são chamados de arquivos estáticos.

Um outro ponto para ser mencionado, é que depois que esses arquivos são acessados pelo navegador ele gera um cache, que basicamente são os arquivos acessados armazenados no computador para não ter que baixar tudo de novo. Caso necessário o navegador faz um refresh apagando o cache e atualizando com um novo.

Dentro do conceito de Client temos uma preocupação que são as diferentes resoluções de tela que devem ser consideradas na criação de algum website. Outra preocupação é a compatibilidade entre navegadores, pois o comportamento de cada navegador é diferente.

E o que são navegadores? Navegadores também chamados de browsers, são programas que são instalados nos dispositivos que tem como objetivo interpretar os arquivos que vem do servidor web, como por exemplo o HTML,CSS, JavaScript, imagens assim como outros tipos de arquivos. E cada navegador tem sua forma de tratar os arquivos que vem do servidor.

Dentro de Client temos mais assunto que se chama aplicação web, que são soluções criadas que possuem a internet como meio de comunicação e que não precisam ser instaladas no dispositivo.

Exemplos: O Youtube acessado pelo navegador é uma aplicação web. Já quando acessamos o mesmo pelo aplicativo deixa de ser uma aplicação web pois usa de outros meios para funcionar.

O que são Servers
(voltar)

Agora vamos entender um pouco sobre os servidores? Servidores nada mais são que computadores que têm um comportamento diferente do Client, também usam a conexão com a internet para fazer suas comunicações, porém não é o único meio de acesso, nesse caso que não necessita da internet chamamos de servidores locais ou rede local.

Os computadores que são usados como servidores precisam ser mais potentes pela demanda que irão fazer. Existem vários tipos de servidores para cada demanda.

Exemplo: Arquivos, Segurança (Firewall), Streaming, E-mail, Web.

O arquivo HTML sempre inicia com uma estrutura básica. Geralmente os editores como o Visual Studio Code já tem atalhos para iniciar os documentos HTMLs com essa estrutura, logo, você não precisa se preocupar em decorá-la, mas é bom que faça para entender melhor como ela funciona.

Falando sobre Tags
(voltar)

Tags são o conjunto de caracteres que formam um elemento, ou seja, quando nos referenciamos à Tag "p" estamos falando disso: <p>.

Existem dois tipos de Tags, as que necessitam de fechamento e as que não necessitam de fechamento.

Atributos das Tags
(voltar)

Atributos são informações que passamos na Tag para que ela se comporte da maneira que a gente espera. Dentro de atributo existem os atributos globais que funcionam em todas as Tags e específicos que são direcionados para cada Tag, através de especificação.

Os atributos possuem nome e um valor, existem atributos que você vai usar praticamente sempre e existem outros que são mais raros.

Listas Ordenadas e Não Ordenadas
(voltar)

Lista ordenada é uma tag que quando referenciamos ela fica da seguinte forma <ol> que necessita de fechamento e que é considerada uma tag pai e que precisa de uma tag filha que referenciada fica desse jeito <li>.

Lista não ordenada é uma tag que quando referenciamos ela fica da seguinte forma <ul> que necessita de fechamento e que é considerada uma tag pai e que precisa de uma tag filha que referenciada fica desse jeito <li>.
